# 密钥验证机制

当前的验证方式是**隐式验证**——通过尝试解密来判断密码是否正确。

## 验证流程 (`auth` 方法)

```
用户输入密码
    ↓
读取存储的加密数据
    ↓
提取: salt + iv + encryptedData
    ↓
PBKDF2(密码 + salt) → 派生密钥
    ↓
AES-GCM 解密
    ↓
┌─────────────────┬─────────────────┐
│   解密成功       │   解密失败       │
│   → 密码正确     │   → 密码错误     │
│   → 保存密码到   │   → 返回 false   │
│     内存         │                 │
└─────────────────┴─────────────────┘
```

## 代码实现

```dart
Future<bool> auth(String password) async {
  // 1. 读取加密数据
  final raw = _storageAdapter!.getString(_storageKey);
  
  // 2. 解析加密载荷
  final payload = EncryptedPayload.fromJson(jsonDecode(raw));
  final salt = base64Decode(payload.salt);
  final iv = base64Decode(payload.iv);
  final encryptedData = base64Decode(payload.encryptedData);

  // 3. 用密码派生密钥
  final derivedKey = await HajimiSecurity.deriveKey(password, salt, iterations);
  
  // 4. 尝试解密
  try {
    final plaintext = await aesGcm.decrypt(..., secretKey: secretKey);
    // 解密成功 = 密码正确
    _password = password;  // 保存到内存
    _unlocked = true;
    return true;
  } catch (e) {
    // 解密失败 = 密码错误
    return false;
  }
}
```

## 为什么这样可行？

| 特性 | 说明 |
|------|------|
| **AES-GCM 完整性验证** | GCM 模式会生成 MAC 标签，如果密钥错误，MAC 验证会失败 |
| **无需存储密码哈希** | 不需要额外存储密码的哈希值进行比对 |
| **安全性高** | 攻击者无法区分"密码错误"和"数据损坏" |

## 总结

验证密钥的方式是：**用输入的密码尝试解密存储的数据，解密成功即密码正确**。这是加密存储系统的标准做法，无需单独存储密码哈希。
