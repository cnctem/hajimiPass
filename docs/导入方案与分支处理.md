# 导入方案与分支处理

## 1. 目标

为备份页提供两类导入能力：

- 设置导入
- 账号信息导入

每类导入均支持两种模式：

- 覆盖导入（overwrite）
- 合并导入（merge）

并在导入过程中完成分支判断（不同数据结构、不同冲突场景）和错误兜底。

## 2. 入口与交互

页面位置：`lib/pages/setting/backup_setting.dart`、`lib/pages/setting/models/backup_settings.dart`

新增入口：

- 导入账号信息
- 导入设置项

每个入口支持 3 种内容来源：

- 从文件导入（macOS 使用 `osascript` 选择 JSON 文件）
- 从剪贴板导入
- 手动粘贴 JSON 文本导入

统一流程：

1. 读取内容
2. 选择导入模式（覆盖/合并）
3. 执行导入
4. Toast 回显导入结果

## 3. 核心服务设计

新增：`lib/utils/export/import_service.dart`

### 3.1 ImportService 职责

- 解析输入 JSON
- 识别账号明文/密文结构
- 密文账号解密
- 分发到设置导入或账号导入逻辑
- 统一返回导入结果对象（成功/失败/需要密码）

### 3.2 返回模型

- `ImportResultData`
- 包含 `result`、`message`、`accountSummary`、`appliedSettingsCount`

## 4. 数据分支处理（核心）

## 4.1 账号导入分支

### A. 数据类型分支

1. 明文账号 JSON：直接 `AccountList.fromJson`
2. 密文账号 JSON：判断包含 `encryptedData/iv/salt/iterations`
   - 无密码：返回 `passwordRequired`
   - 有密码：AES-GCM 解密后反序列化 `AccountList`

### B. 导入模式分支

账号底层执行在：`lib/utils/storage/hajimi_storage.dart#importAccounts`

- 覆盖导入
  - 直接替换本地账号列表
  - 标签重建（导入标签 + 账号内标签）并去重

- 合并导入
  - 以 `name` 为主键匹配本地账号
  - 本地不存在：新增
  - 本地存在：比较 `lastEditTime`
    - 导入时间更新或相同：覆盖本地该账号
    - 导入时间更旧：跳过

### C. 结果统计分支

返回 `AccountImportSummary`：

- importedCount
- addedCount
- updatedCount
- skippedCount
- overwrite

用于前端提示“新增/更新/跳过”。

## 4.2 设置导入分支

### A. 数据结构分支

- 若存在 `settings` 字段且为 Map：使用 `settings` 子对象
- 若包含 `accountList` 字段：判定为账号数据，拒绝设置导入
- 否则按“纯设置 JSON”处理

### B. 导入模式分支

设置底层执行在：`lib/utils/storage/storage.dart`

- 覆盖导入：`replaceAll()`
- 合并导入：`putAll()`

### C. 类型规范化分支

为避免脏数据污染，按 key 做类型收敛（如 bool/int/double/string）：

- `themeMode`、`schemeVariant`、`customColor` 等数值字段强制转换
- `isPureBlackTheme`、`dynamicColor` 等布尔字段严格校验
- 非法值忽略，不写入

导入设置后刷新 `ThemeController` 的运行时状态，保证 UI 即时生效。

## 5. 异常与兜底策略

- JSON 解析失败：返回“导入文件格式错误”
- 未解锁账号仓库：返回“请先解锁账号数据”
- 密文账号缺密码：返回 `passwordRequired`
- 密码错误/解密失败：返回导入失败信息
- 内容为空：前端直接拦截提示

## 7. 分支处理解决方法总结

本次导入的“分支处理”可归纳为三层：

1. **数据结构分支**：先判断导入内容类型（设置/账号、明文/密文）
2. **业务策略分支**：按覆盖或合并执行不同写入策略
3. **冲突分支**：合并时以 `name + lastEditTime` 决策更新或跳过

该方案的好处：

- 规则稳定，可预测
- 兼容历史导出格式和纯设置 JSON
- 出错时可定位（结构错、密码错、类型错）
